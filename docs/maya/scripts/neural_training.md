# Neural Training

The Neural Training script is a Python script designed to run the AdonisML training process from a standalone Python entry point or from a command-line interface.

> [!NOTE]
> This script is DCC independent. It can be run from a standalone environment or wrapped by DCC-specific tools, as long as the required AdonisML dependencies, license, and input data are available.

> [!NOTE]
> Neural training is only supported with the **AdonisML** bundle. Make sure the Adonis plugin is installed and licensed to support ML before running the training workflow.

The script trains an `.adnm` model from the input and output `.csv` files generated by the data extraction workflow. It can also use neural cluster data generated by an AdonisML neural clustering workflow to provide locality information during training.

The training script can be launched directly from Python using the `train` function in `adnml.scripts.train`, or through a command-line interface using the supported command-line arguments.

DCC-specific tools may also wrap this same training process and provide UI-based workflows to configure and launch training interactively.

The training script checks out an Adonis license while the training process is running. The license product depends on the execution mode. Standalone or automated training usually uses the `"batch"` license product, while DCC-driven interactive execution can use the `"interactive"` license product.

The main Python function to launch training is `train`, which is defined as follows:

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">from adnml.scripts.train import train

def train(
    in_filepath,                         # str: filepath to the input csv data.
    out_filepath,                        # str: filepath to the output csv data.
    model_target_path,                   # str: path where the trained model will be saved.
    clusters_filepath="",                # str: optional path to the neural cluster json file.
    joint_info_filepath="",              # str: optional path to the recorded joint info json file.
    aug_protocol=AugmentationProtocols.NONE, # str: data augmentation protocol.
    force_wait=True,                     # bool: whether to wait for input before closing the terminal.
    standardize_in=False,                # bool: whether to standardize input data.
    standardize_out=False,               # bool: whether to standardize output data.
    use_only_aug=False,                  # bool: whether to train using only augmented data.
    force_overwrite=False,               # bool: whether to overwrite the model file if it exists.
    license_product="batch"              # str: Adonis license product type.
)
</code></pre>

The `train` function launches the training process using the provided training data, output model path, and optional training settings.

> [!NOTE]
> This page describes the standalone Neural Training script and the `adnml.scripts.train` Python entry point.

## Requirements

To run the Neural Training script, the following files are required:

- `in_filepath`: Input data `.csv` file used for training.
- `out_filepath`: Output data `.csv` file used for training.
- `model_target_path`: Output `.adnm` model path.

The input and output data files are usually generated by the data extraction workflow.

The output model path must use the `.adnm` extension.

When training with neural clusters, the following additional files are required:

- `clusters_filepath`: Neural cluster `.json` file generated by an AdonisML neural clustering workflow.
- `joint_info_filepath`: Joint information `.json` file generated during data extraction. This file should be selected from the same dataset folder as the input and output data files generated during data extraction.

> [!NOTE]
> If `clusters_filepath` is provided, `joint_info_filepath` should also be provided so the training process can associate the neural clusters with the recorded joint information.

## Training Dependencies

The Neural Training script requires the AdonisML training dependencies to be installed before training can run.

To satisfy the requirements for GPU training, run the provided dependency installation script for your platform:

- `install_dependencies.bat` on Windows.
- `install_dependencies.sh` on Linux.

These scripts install the dependencies required for GPU execution.

When GPU-capable ML dependencies and a compatible GPU are available, training uses the GPU. GPU training is usually much faster than CPU training.

If GPU execution is not available, the training process falls back to CPU execution. CPU training can be significantly slower, especially for larger datasets or more complex training configurations.

## Arguments

In this section we provide a brief overview of the arguments of the `train` function.

| Argument | Required | Type | Default | Description |
| :------- | :------- | :--- | :------ | :---------- |
| `in_filepath` | Yes | string |  | Path to the input data `.csv` file used for training. |
| `out_filepath` | Yes | string |  | Path to the output data `.csv` file used for training. |
| `model_target_path` | Yes | string |  | Path where the trained model will be saved. The output file must use the `.adnm` extension. |
| `clusters_filepath` | Optional | string | `""` | Path to the neural cluster `.json` file containing cluster data. This file is generated by an AdonisML neural clustering workflow. |
| `joint_info_filepath` | Optional | string | `""` | Path to the recorded joint information `.json` file exported with the extracted data. Required when `clusters_filepath` is provided. |
| `aug_protocol` | Optional | string | `AugmentationProtocols.NONE` | Data augmentation protocol to use. Available values are `"none"`, `"simple"`, and `"random"`. |
| `force_wait` | Optional | boolean | `True` | Whether to force the terminal to wait for user input before closing. |
| `standardize_in` | Optional | boolean | `False` | Whether to standardize the input data before training. |
| `standardize_out` | Optional | boolean | `False` | Whether to standardize the output data before training. This is generally recommended for most training runs. |
| `use_only_aug` | Optional | boolean | `False` | Whether to train using only augmented data, without including the original samples. |
| `force_overwrite` | Optional | boolean | `False` | Forces overwrite of the output model file if it already exists. |
| `license_product` | Optional | string | `"batch"` | Type of Adonis product license to check. Supported values are `"batch"` and `"interactive"`. |

## How To Use

1. Install the training dependencies.

    To satisfy the requirements for GPU training, run the dependency installation script for your platform.

    On Windows, run:

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">install_dependencies.bat
</code></pre>

    On Linux, run:

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">install_dependencies.sh
</code></pre>

    When GPU-capable dependencies and a compatible GPU are available, training uses the GPU. Otherwise, the training process falls back to CPU execution.

2. Generate the training dataset.

    Generate the input and output `.csv` files using the data extraction workflow.

3. Create the required file paths.

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">input_data = "path/to/dataset/inputs.csv"
output_data = "path/to/dataset/outputs.csv"
model_path = "path/to/output/model.adnm"
</code></pre>

4. Run training from Python.

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">from adnml.scripts.train import train

train(
    in_filepath=input_data,
    out_filepath=output_data,
    model_target_path=model_path,
    standardize_out=True
)
</code></pre>

5. Optionally train with neural clusters.

    Neural clusters can be used to provide locality information during training.

    The neural cluster `.json` file is generated by an AdonisML neural clustering workflow.

    The joint information `.json` file should come from the same dataset folder as the input and output data files generated during data extraction.

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">from adnml.scripts.train import train
from adnml.utils.constants import AugmentationProtocols

train(
    in_filepath="path/to/dataset/inputs.csv",
    out_filepath="path/to/dataset/outputs.csv",
    model_target_path="path/to/output/model.adnm",
    clusters_filepath="path/to/clusters/neural_clusters.json",
    joint_info_filepath="path/to/dataset/joints.json",
    aug_protocol=AugmentationProtocols.NONE,
    standardize_out=True
)
</code></pre>

6. Optionally enable augmentation.

    Augmentation is experimental and should only be used when training without augmentation is not giving good results and the dataset is small.

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">from adnml.scripts.train import train
from adnml.utils.constants import AugmentationProtocols

train(
    in_filepath="path/to/dataset/inputs.csv",
    out_filepath="path/to/dataset/outputs.csv",
    model_target_path="path/to/output/model.adnm",
    clusters_filepath="path/to/clusters/neural_clusters.json",
    joint_info_filepath="path/to/dataset/joints.json",
    aug_protocol=AugmentationProtocols.SIMPLE,
    standardize_out=True
)
</code></pre>

7. Optionally overwrite an existing model.

    By default, training does not overwrite an existing model file.

    Enable `force_overwrite` only if the existing `.adnm` file can be safely replaced.

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">from adnml.scripts.train import train

train(
    in_filepath="path/to/dataset/inputs.csv",
    out_filepath="path/to/dataset/outputs.csv",
    model_target_path="path/to/output/model.adnm",
    standardize_out=True,
    force_overwrite=True
)
</code></pre>

</code></pre>

> The exact executable or launcher used to run the command-line arguments depends on the installation and execution context. The Python `train` function builds the command internally using the AdonisML runner.

## Standardization

The script supports standardization for both input and output data.

- `standardize_in` or `--standardize-in`: Standardizes input data.
- `standardize_out` or `--standardize-out`: Standardizes output data.

For most training runs, output standardization is recommended because it can improve training stability and results.

If the number of training poses is low, it may be better to disable output standardization to preserve the original data distribution.

## Neural Clusters

Neural cluster data can be provided with `clusters_filepath` or `--clustersfilepath`.

When neural clusters are used, the training process also requires the joint information file through `joint_info_filepath` or `--joint-info-filepath`.

The neural cluster `.json` file is generated by an AdonisML neural clustering workflow. It stores painted cluster regions, cluster names, joint associations, and per-component weights.

The joint information `.json` file is exported during data extraction and should come from the same dataset folder as the input and output data files.

## Augmentation

The `aug_protocol` argument controls which data augmentation protocol is used.

The available protocols are:

- `"none"`: No data augmentation is applied.
- `"simple"`: Composes augmented poses using the current pose for clustered vertices and the rest pose, which corresponds to sample index `0`, for the remaining samples.
- `"random"`: Composes augmented poses using the current pose for clustered vertices and a random pose from the provided samples.

Augmentation uses neural clusters to generate new synthetic poses from the training samples. These generated poses may contain artifacts and may be less aligned to the original simulated silhouette.

The `use_only_aug` or `--use-only-aug` option can be used to train only with the augmented synthetic data, without including the original samples.

> [!NOTE]
> `use_only_aug` is ignored when no augmentation protocol is selected.

## License Product

The `license_product` argument defines the Adonis license product type to check during training.

Supported values are:

- `"batch"`
- `"interactive"`

The default value is `"batch"`.

Depending on the execution mode, training will consume either a batch license or an interactive license.

Use `"batch"` for standalone, automated, or non-interactive training workflows. Use `"interactive"` when the training process is launched from an interactive DCC workflow and should check against an interactive Adonis license.

## Force Wait

The `force_wait` argument and `--force-wait` command-line flag force the terminal to wait for user input before closing.

This is useful when the training process is launched from a temporary terminal window and the user needs time to read the output before the terminal closes.

When `--force-wait` is enabled, the script waits for input with the following message:

<pre><code style="white-space: pre; margin: 20px 0; padding: 10px; box-sizing: border-box;">Press any key to close the terminal...
</code></pre>

## Output Files

After a successful training run, the following files are generated:

- `<model_name>.adnm`: Trained Adonis neural model file.
- `<model_name>_log.txt`: Training log file generated next to the `.adnm` model.
- `<model_name>_config.json`: Configuration file storing the training parameters used for the run.

The `<model_name>_log.txt` file can be used to inspect the training process. It contains epoch information, average loss, best loss, warnings, errors, and other training messages.

The `<model_name>_config.json` file can be used to review the training configuration used to produce the model.

## Troubleshooting

If the training process fails or does not produce the expected result, check the following:

1. Check the generated `<model_name>_log.txt` file.

    The log file is generated next to the `.adnm` model and contains detailed training information, including epochs, losses, warnings, and errors.

2. Check the generated `<model_name>_config.json` file.

    The config file records the training parameters used for the run. This can help confirm that the correct input data, output data, neural cluster settings, augmentation settings, license product, and output options were used.

3. Confirm that the model path is valid.

    The output model path must use the `.adnm` extension.

4. Confirm that the input and output data paths are valid.

    The `in_filepath` and `out_filepath` arguments must point to valid `.csv` files.

5. Confirm that neural cluster paths are valid when using neural clusters.

    If `clusters_filepath` is provided, make sure `joint_info_filepath` is also provided and points to the joint information `.json` file from the extracted dataset.

6. Confirm that the training dependencies are installed.

    To satisfy the requirements for GPU training, run `install_dependencies.bat` on Windows or `install_dependencies.sh` on Linux.

    If GPU execution is not available, the training process falls back to CPU execution, which can be significantly slower.

7. Confirm the license product being used.

    Use `"batch"` for standalone or automated training workflows. Use `"interactive"` when the training process is launched from an interactive DCC workflow.

## Result

As a result of executing the script, the output folder will contain the trained model and the training support files:

- `<model_name>.adnm`
- `<model_name>_log.txt`
- `<model_name>_config.json`

The resulting `.adnm` file can then be used as part of the Adonis neural workflow.

## Recommendations

- Run `install_dependencies.bat` on Windows or `install_dependencies.sh` on Linux before training if GPU training is required.
- Use data generated by the data extraction workflow as the training input and output data.
- Keep the input data, output data, and joint info files together in the same dataset folder.
- Use `standardize_out` or `--standardize-out` for most training runs.
- Consider disabling output standardization only when the number of training poses is low and preserving the original output distribution gives better results.
- Use neural clusters when local deformation regions should be isolated during training.
- Use augmentation only when the dataset is small and training without augmentation is not producing good results.
- Use a compatible GPU when available, because GPU training is usually much faster than CPU training.
- Use `"batch"` licensing for standalone or automated training workflows, and `"interactive"` licensing for DCC-driven interactive workflows.
- Review the generated `<model_name>_log.txt` file after training to inspect epoch progress and loss values.
- Review the generated `<model_name>_config.json` file to confirm the training configuration used for the model.
- Use `force_overwrite` or `--force-overwrite` carefully, because it deletes any existing model file at the target path.

## Limitations

- Augmentation is experimental and should only be used when training without augmentation is not giving good results and the dataset is small.
- Augmented poses may contain artifacts and may be less aligned to the original simulated silhouette.
- Neural cluster training requires a valid neural cluster `.json` file and a valid joint information `.json` file.
- Existing model files will prevent training unless `force_overwrite` or `--force-overwrite` is enabled.
